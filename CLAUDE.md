# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Go web app for competition operations. Echo v5 backend, HTMX/Alpine.js + Go templates frontend, PostgreSQL database. Runtime config loaded from a persisted JSON file with environment overrides.

## Commands

```bash
go build ./cmd/server            # Build web server
go build ./cmd/db                # Build database CLI
go run ./cmd/server              # Start server
go run ./cmd/db                  # Database CLI (init-config, seed-admin, jwt-key)
go test ./...                    # Run all tests
go test ./internal/service/...   # Run tests for a single package
sqlc generate                    # Regenerate DB code after editing db/queries/*.sql
atlas migrate diff <name> --env local   # Generate migration after editing db/schema.sql
atlas migrate apply --url "$DATABASE_URL"  # Apply pending migrations
atlas migrate lint --env local --latest 1  # Lint latest migration
```

## Critical Rules

- **Echo v5, not v4.** Import `github.com/labstack/echo/v5`. Handlers take `*echo.Context` (pointer, not interface).
- **No `database/sql`.** All DB access via sqlc-generated code backed by `pgx/v5`.
- **No raw SQL in Go.** Queries go in `db/queries/*.sql`, then run `sqlc generate`.
- **Never hand-edit `db/sqlc/`.** Generated by sqlc; will be overwritten.
- **Never edit applied migrations.** Edit `db/schema.sql` first, then `atlas migrate diff`. Never modify `atlas.sum` manually.
- **Never hardcode secrets.** JWT keys, passwords, and database URLs come from config/environment.

## Architecture

**Layered design:** Handlers (`internal/handler/`) -> Service (`internal/service/`) -> Database (`db/sqlc/` generated code). Handlers are thin; business logic lives in the service layer.

**Four handler groups**, each a struct receiving service dependencies via constructor:

| Handler | Prefix | Auth | Purpose |
|---------|--------|------|---------|
| `PublicHandler` | `/` | No | Landing page |
| `AuthHandler` | `/` | No | Register, sign-in, sign-out, refresh |
| `AppHandler` | `/app` | Yes | Profile (`/app` redirects to `/app/profile`) |
| `AdminHandler` | `/app/admin` | Yes + admin | User CRUD |

**Auth flow:** JWT access + refresh tokens stored as HTTP-only cookies. Database-backed signing keys with lifecycle management (active/retired/revoked). `Viewer` loaded by `loadCurrentViewer` middleware, retrieved via `auth.CurrentViewer(c)`. Database roles: `user`, `admin`, enforced by `requireRole` middleware. Unauthenticated sessions get a synthetic `guest` sentinel viewer so role checks never need nil guards.

**Templates:** `.gohtml` files. Layouts in `web/templates/layouts/`, pages in `web/templates/pages/`, partials in `web/components/`. Template names (e.g. `"app/profile"`) map to layout+page pairs registered in `internal/server/render.go`. Custom funcs: `hasRole`, `hasPrefix`, `join`. HTMX for server interactions, Alpine.js for client-side interactivity.

**Config:** JSON file (default `data/var/drynn/server.json`) with env overrides. Written by `cmd/db init-config`. Key env vars: `DATABASE_URL`, `APP_ADDR`, `HOBO_CONFIG_PATH`, `JWT_ACCESS_TTL`, `JWT_REFRESH_TTL`, `COOKIE_SECURE`.

## Code Style

- Standard Go: `gofmt`, `go vet`, short names in tight scopes.
- Wrap errors: `fmt.Errorf("context: %w", err)`.
- Pass `context.Context` through service and DB calls.
- Domain errors defined in `internal/service/errors.go`.
- View data structs embed `BaseViewData` (title, path, flash, current user) in `handler/view.go`.

## Database

- **sqlc config:** `sqlc.yaml` maps `db/queries/*.sql` -> `db/sqlc/` with pgx/v5. Type overrides: `uuid` -> `uuid.UUID`, `timestamptz` -> `time.Time`.
- **Schema:** `db/schema.sql` is the Atlas source of truth. Tables: `roles`, `users`, `user_roles`, `jwt_signing_keys`.
- **Constraints:** Handles are lowercase `^[a-z0-9_]+$` (3-32 chars), emails lowercase, both unique. UUID primary keys via `gen_random_uuid()`.
