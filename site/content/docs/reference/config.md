The server reads configuration from a JSON file, then applies
environment variable overrides. Environment variables always take
precedence over the file.

## Config file

Default path: `data/var/drynn/server.json` (override with
`DRYNN_CONFIG_PATH` env var). Generated by `cmd/db init-config`.

The file uses version `1`. Unknown fields cause a startup error.

## Fields

| JSON field | Env variable | Type | Default | Description |
|------------|-------------|------|---------|-------------|
| `app_addr` | `APP_ADDR` | string | `:8080` | Listen address (host:port) |
| `database_url` | `DATABASE_URL` | string | *(required)* | PostgreSQL connection string |
| `data_dir` | `DATA_DIR` | string | `data/var/drynn/data` | Server-managed data directory |
| `jwt_access_ttl` | `JWT_ACCESS_TTL` | duration | `15m` | Access token lifetime |
| `jwt_refresh_ttl` | `JWT_REFRESH_TTL` | duration | `168h` | Refresh token lifetime (7 days) |
| `cookie_secure` | `COOKIE_SECURE` | bool | `false` | Set `Secure` flag on auth cookies |
| `mailgun.api_key` | `MAILGUN_API_KEY` | string | *(empty)* | Mailgun private API key |
| `mailgun.sending_domain` | `MAILGUN_SENDING_DOMAIN` | string | *(empty)* | Mailgun sending domain |
| `mailgun.from_address` | `MAILGUN_FROM_ADDRESS` | string | *(empty)* | Sender email address |
| `mailgun.from_name` | `MAILGUN_FROM_NAME` | string | *(empty)* | Sender display name |
| `request_access_enabled` | `REQUEST_ACCESS_ENABLED` | bool | `false` | Enable `/request-access` form |
| `admin_contact_email` | `ADMIN_CONTACT_EMAIL` | string | *(empty)* | Destination for access requests |

## Loading order

1. Hardcoded defaults are set.
2. JSON file is read and merged (non-empty fields override defaults).
3. Environment variables are applied (non-empty values override file).
4. `database_url` is validated — startup fails if empty.

## Type parsing

- **Strings:** Used as-is when the env var is non-empty.
- **Durations:** Parsed with Go's `time.ParseDuration` (e.g. `30m`,
  `2h`, `168h`). Invalid values silently fall back to the file
  value.
- **Bools:** Parsed with `strconv.ParseBool` — accepts `true`,
  `false`, `1`, `0`, `yes`, `no`, `on`, `off`. Invalid values fall
  back.

## Mailgun

Email features (invitations, password resets, access requests) are
only active when Mailgun is configured. All three of `api_key`,
`sending_domain`, and `from_address` must be non-empty for
`Configured()` to return true.

## Example

```json
{
  "version": 1,
  "app_addr": ":8080",
  "database_url": "postgres://drynn:secret@localhost:5432/drynn?sslmode=disable",
  "data_dir": "data/var/drynn/data",
  "jwt_access_ttl": "15m0s",
  "jwt_refresh_ttl": "168h0m0s",
  "cookie_secure": false,
  "mailgun": {
    "api_key": "",
    "sending_domain": "",
    "from_address": "",
    "from_name": ""
  },
  "request_access_enabled": false,
  "admin_contact_email": ""
}
```
